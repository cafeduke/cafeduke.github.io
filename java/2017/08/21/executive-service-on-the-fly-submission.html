Executor service - On the fly submission

In this article, we shall recursively navigate through a given directory using an ExecutorService. Multiple threads shall involve in navigating directories and analyzing files simultaneously. There are two executor services serviceDir and serviceFile. The  ExecutorService serviceDir shall spawn threads to dig into directories in parallel. The ExecutorService serviceFile shall spawn threads to calculate the size of file in parallel.

DirTask - A Callable

Each thread executing a DirTask shall list contents under the current directory

  • A Callable task (DirTask) is created for each child directory under the current directory.

  • The callable task is submitted using the executor service.

  • So, another thread in the ExecutorService (serviceDir) shall take care of digging through this child directory while the current thread goes ahead analyzing further files in the current directory.

  • A Callable task (FileTask) is created for each file under the current directory.

  • The callable task is submitted using the executor service.
  • So, another thread from another ExecutorService (serviceFile) shall take care of analyzing the file while the current thread goes ahead add adds the Future result object into a BlockingQueue.
  • Please note that the result object is added into BlockingQueue. The thread does not wait for the file size calculation to complete. We are just adding the Future object using which we can obtain the result at a later point in time.
/**  
 * Analyze the list of files in the current directory.   
 * <ul>  
 *    <li>If the file is a directory, </li>   
 * </ul>  
 */  
private class DirTask implements Callable<Void>  
{  
   private File dir;  
     
   public DirTask (File dir)  
   {  
      this.dir = dir;  
   }  
     
   @Override  
   public Void call() throws Exception  
   {  
      for (File file : dir.listFiles())  
      {  
         if (file.isDirectory())  
         {  
            Util.threadLog("Submit DirTask. Dir=" \+ pathBaseDir.relativize(file.toPath()));  
            serviceDir.submit(new DirTask(file));  
         }  
         else  
         {  
            Util.threadLog("Submit FileTask. File=" \+ pathBaseDir.relativize(file.toPath()));  
            Future<Long> result = serviceFile.submit(new FileTask(file));  
            queueResult.put(result);  
         }  
      }  
      return null;  
   }        
}

FileTask - A Callable

Each thread calculates and returns the size of a file.

private class FileTask implements Callable<Long>  
{  
   private File file;       
     
   public FileTask (File file)  
   {  
      this.file = file;  
   }  
     
   @Override  
   public Long call() throws Exception  
   {  
      Util.threadLog("File=" \+ pathBaseDir.relativize(file.toPath()) + " Size=" \+ formatSize(file.length()));  
      Util.sleepInMilli(10);  
      return file.length();  
   }  
}

Recursive calculation of directory size

The recursive directory size calculation proceeds as follows

  • A DirTask is created using the base directory and submitted using ExecutorService serviceDir.
  • This shall result in various threads from serviceDir ExecutorService to dig recursively into directories. As a result further submissions of DirTask using serviceDir and FileTask using serviceFile ExecutorService shall take place.
  • By specifying the thread pool size of serviceDir and serviceFile we control on the number of threads that should involve in digging into directories and number of threads that should analyze each file to get the file size.
  • The main thread after triggering the recursive navigation shall poll the BlockingQueue for Future result objects
  • Polling a queue utilizes the built in mechanism of waiting for objects to appear and removes the first element in the queue when it appears.
  • The result objects are used to Future.get the value returned  by FileTask. Thus obtained value is aggregated to calculate the total sum.
public class ExecutorService_ProducerConsumer  
{     
   private BlockingQueue<Future<Long>\> queueResult = new ArrayBlockingQueue<>(10);  
     
   private ExecutorService serviceDir;  
     
   private ExecutorService serviceFile;  
     
   private Path pathBaseDir;  
     
   public static void main (String arg[]) throws Exception  
   {  
      ExecutorService_ProducerConsumer demo = new ExecutorService_ProducerConsumer ();  
      long size = demo.getSize(new File("D:\\\Media\\\Movies"));  
      System.out.println("------------------------------------------------------------------------");  
      System.out.println("Total Size = " \+ formatSize(size));  
      demo.clean();  
   }  
     
   public ExecutorService_ProducerConsumer ()  
   {  
      serviceDir = Executors.newFixedThreadPool(2);  
      serviceFile = Executors.newFixedThreadPool(3);        
   }  
     
   public long getSize (File file) throws Exception  
   {  
      if (file == null || !file.exists())  
         throw new FileNotFoundException("File does not exist. File=" \+ file);  
        
      if (file.isFile())  
         return file.length();  
        
      pathBaseDir = file.toPath();  
      queueResult.clear();  
        
      serviceDir.submit(new DirTask(file));        
      Future<Long\> result = null;  
      long sum = 0;  
      while ((result = queueResult.poll(3, TimeUnit.SECONDS)) != null)  
         sum += result.get();  
        
      return sum;  
   }  
     
   public void clean()  
   {  
      serviceDir.shutdown();  
      serviceFile.shutdown();       
   }   
  
   ...  
   ...  
}
